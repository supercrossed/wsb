How It Works

The Premise

r/wallstreetbets is (almost) always wrong. This tool monitors WSB sentiment in real time and recommends the inverse of whatever the crowd is feeling. If WSB is bullish, we say PUTS. If bearish, CALLS. The trade bot can then automatically execute 0DTE SPY options based on that signal.

Data Collection

  • Reddit polling — Every 60 seconds, the app fetches comments from the active WSB discussion thread (Daily, Overnight, or Weekend) plus the top 10 hot posts and their comments, using Reddit's public JSON API (no auth required).
  • Thread schedule — Daily Discussion (7 AM–4 PM EST weekdays), What Are Your Moves Tomorrow (4 PM–7 AM weekdays), Weekend thread (Fri 4 PM–Sun 4 PM). On Sunday at 4 PM, both weekend and overnight threads are dual-polled so no comments are missed.
  • Comment storage — Comments are stored in a local SQLite database with 2-day retention. Each comment is tagged with its thread type, creation time, upvote count, and analyzed sentiment.
  • SPY prices — Daily SPY data is fetched from Yahoo Finance's public chart API. A 90-day backfill runs on startup, and a daily refresh runs at 5 PM EST after market close.

Sentiment Analysis

Each comment is run through a seven-layer sentiment pipeline:

  • Layer 1 NLP analysis — The sentiment library scores full sentences for positive/negative tone.
  • Layer 2 WSB phrase patterns — 60+ context-aware patterns match WSB-specific language (e.g., "my puts about to rip" = bearish, "bers r fuk" = bullish, "bols r fuk" = bearish).
  • Layer 3 Degen slang — Recognizes bers/bols/buls misspellings, thundercock, joever, cooked, and mocking patterns with context-aware scoring.
  • Layer 4 Emoji scoring — Rockets and moon emojis score bullish; bear and skull emojis score bearish; diamond hands score bullish.
  • Layer 5 Sarcasm detection — 16 patterns (e.g., "what could go wrong", "this is fine", "/s") invert weak signals and discard ambiguous ones.
  • Layer 6 Temporal awareness — 10 past-tense patterns (e.g., "expired worthless", "was up 200%") discount the signal by 70%, since past events don't predict the future.
  • Layer 7 Question & relevance gating — Questions that aren't directional opinions are dampened. Non-financial NLP-only comments are discarded entirely to avoid noise.

Upvote Weighting

Comments are upvote-weighted — a comment with 50 upvotes counts 50x more than one with 1. This lets the community's own voting amplify signal and suppress noise. The dashboard shows both the raw comment count and the upvote-weighted score.

Inverse Recommendation

After sentiment is aggregated, bullish and bearish percentages are compared (neutral excluded). If the spread exceeds 5%, the inverse recommendation fires:

  • WSB is bullish → Recommend PUTS
  • WSB is bearish → Recommend CALLS
  • Spread under 5% → HOLD

Accuracy is tracked daily by comparing each inverse recommendation against actual SPY movement at close.

Trade Bot

Overview

The trade bot automatically executes 0DTE (zero days to expiration) SPY options trades on Alpaca based on the sentiment signal. It supports two modes: Inverse (bet against WSB) and WSB (follow the crowd). Both paper and live trading are supported.

Entry Logic

  • Market open evaluation — At 9:30 AM EST, the bot evaluates time-decay weighted sentiment from the past 48 hours. Comments closer to market open count more: 1.0x within 2.5h, 0.7x at 2.5–5.5h, 0.5x at 5.5–17.5h, and 0.3x beyond.
  • Retry on HOLD — If the sentiment spread is under 5% (HOLD), the bot retries at 10:00 AM, 10:30 AM, and 11:00 AM. After 11:00 AM, theta decay makes 0DTE entry too risky, so the bot skips the day.
  • Option selection — The bot picks 0DTE SPY contracts in the ATM to 1.5% OTM zone, optimizing for the contract that maximizes (number of contracts) × (proximity to ATM).
  • Position sizing — Based on the configured risk level: Safe 30% Degen 50% YOLO 70% of total portfolio equity is allocated per trade.

Exit Logic

  • Profit target — The position is closed when the option price rises 10% above entry.
  • Stop loss — Hard stop at 20% below entry price. If the option drops 20%, the position is immediately closed.
  • Trailing stop — Once profit exceeds 10%, the bot switches to a trailing stop set at 50% of unrealized profit. For example, if the option is up 30%, the trailing stop triggers at +15%. This lets winners run while locking in gains.
  • End of day — At 3:45 PM EST, any remaining open position is closed to avoid expiration risk.
  • Monitoring — Open positions are checked every 1 second for exit conditions.

Execution

  • Trades are placed via the Alpaca Markets API using market orders for options.
  • Each trade is logged to the database with entry/exit prices, P&L, and the sentiment signal that triggered it.
  • The bot resets daily — the trade tracker clears at market open, allowing one trade per day per bot.
